# 🧭 yaruki - 行動ナビゲーション

Yes/No だけで「次の一歩」が見える行動ナビゲーションアプリケーション。

**Live Demo:** https://s4na.github.io/yaruki/

---

## 📋 概要

「やりたいことはあるけど、何から始めたらいいかわからない」という悩みを、7つのYes/No質問で解決します。

あなたの回答パターンに基づいて、**行動をブロックしている本当の理由を特定**し、具体的な「次の一歩」を提案します。

---

## ✨ 主な特徴

### 🎯 シンプルな質問フロー
- 質問は「Yes」または「No」だけ
- 7つのステップで行動のボトルネックを分析
- 難しい診断ではなく、実践的な「次のアクション」を提案

### 📊 3つのページで構成
1. **質問ページ** - Yes/No形式で進むインタラクティブなクイズ
2. **結果ページ** - 分析結果と具体的な行動提案を表示
3. **回答経路ページ** - 自分の選択肢と判断の流れを可視化

### 💾 ページ内での状態管理
- 質問⇔結果⇔回答経路を同一ページ内で切り替え
- LocalStorageで回答履歴を保持
- スムーズなページ遷移とアニメーション

---

## 🚀 使用方法

### 基本的な操作フロー

```
1. 質問に答える
   ↓
2. Yes/Noボタンをクリック
   ↓
3. 最後の質問に答えると、自動的に結果ページへ遷移
   ↓
4. 「あなたの回答経路を見る」で選択肢を確認
   ↓
5. 「別の視点で見直す」で新たに診断を開始
```

### ページの切り替え

- **質問ページ** → **結果ページ**: 全ての質問に回答後、自動遷移
- **結果ページ** → **回答経路ページ**: 「あなたの回答経路を見る」ボタン
- **回答経路ページ** → **結果ページ**: 「結果に戻る」ボタン
- **いずれのページからでも** → **質問ページ**: 「別の視点で見直す」ボタン

## 🔄 ページ遷移の仕組み

```
index.html
├─ 質問セクション（.question-section）
│   └─ Yes/Noボタンで回答を進める
│
├─ 結果セクション（.result-section）
│   └─ 行動提案を表示（hidden状態で初期化）
│
└─ 回答経路セクション（.journey-section-page）
    └─ 全ての回答を一覧表示（hidden状態で初期化）
```

JavaScript（script.js）で `hidden` クラスの追加・削除により、セクション間の切り替えを実現しています。

## 💡 質問の構成

1. **やりたいことは具体的ですか？** - 目標の明確さを確認
2. **やりたい理由がハッキリしていますか？** - 動機の強さを確認
3. **やろうと思った瞬間に止まることはありますか？** - ブロッカーの有無を確認
4. **止まる原因は『準備が重くて動けない』ことですか？** - 準備のコストを確認
5. **止まる原因は『タスクが大きくて手がつけられない』ことですか？** - タスク規模を確認
6. **止まる原因は『気持ちの問題（不安・だるい・重い）』ですか？** - メンタルブロッカーを確認
7. **周囲の環境が邪魔になっていますか？** - 環境要因を確認

## 🛠️ 開発

### セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/s4na/yaruki.git
cd yaruki

# ローカルで実行
python -m http.server 8000
# または
npx http-server
```

ブラウザで `http://localhost:8000` にアクセスしてください。

### テスト

```bash
npm test
```
