# 🧭 **Yes / No だけで行動が見える 質問フロー（決定木）**

下の7ステップが順に Yes/No で進む構造だよ。
これをそのままサイトの質問として並べれば、
**回答に応じて"行動プラン"が分岐しながら生成される仕組み**が作れる。

---

## **STEP 1｜やりたいことは"具体的"ですか？**

Q1. 「やりたいこと」は具体的に言葉にできていますか？
- Yes → STEP2
- No → **行動：30秒で"できる形"に言語化する**

（例）

* 漠然：運動したい → 具体：3分歩く
* 漠然：勉強したい → 具体：教材を開く

---

## **STEP 2｜やりたい理由がハッキリしていますか？**

Q2. なぜそれをやりたいか、理由（価値）が分かっていますか？
- Yes → STEP3
- No → **行動：理由を1つだけ書く（深掘り不要）**

理由が弱いと行動の持続が落ちるので、ここは重要。

---

## **STEP 3｜始めようとするとき、どこで止まりますか？**

Q3. 「やろうと思った瞬間に止まる」ことはありますか？
- Yes → STEP4
- No → STEP5 へ

---

## **STEP 4｜止まる原因は"準備の重さ"ですか？**

Q4. 準備が重くて動けないと感じますか？
- Yes → **行動：準備を分解し"一番最初の動作"をやる**

例：

* 走る → 靴を出す
* 勉強 → 机に座る
* 掃除 → ゴミ袋を取り出す

- No → 次の質問へ

---

## **STEP 5｜止まる原因は"タスクが大きいこと"ですか？**

Q5. タスクそのものが大きくて手がつけられないですか？
- Yes → **行動：タスクを1/10にする（1分単位に縮小）**
- No → STEP6 へ

---

## **STEP 6｜止まる原因は"気持ち（不安・だるさ）"ですか？**

Q6. 気持ちの問題（不安、だるい、重い）が理由ですか？
- Yes → **行動：行動後の"ご褒美"を設定する or 1分だけやる**
- No → STEP7

---

## **STEP 7｜環境が邪魔していませんか？**

Q7. 周囲の環境が邪魔になっていませんか？（散らかり・通知・騒音など）
- Yes → **行動：環境を1つだけ調整する**
- No → **行動：時間と場所を"固定"して小さく始める**

---

## 🌟 **STEP 8｜最後に決める「今日の一歩」**

Q8. 今日、1分でもできる最小の一歩が決まりましたか？
- Yes → **実行！**
- No → 必要なら STEP4 に戻る

---

## 🧩 Web 実装方針

ボタンは「Yes」「No」だけ。

```
質問 → Yes → 次の質問
        No  → 行動提案（テキスト表示）して終了 or 次へ
```

各質問ごとに

* Yes の場合の遷移先
* No の場合の行動提案
  を持てば、シンプルな **決定木 JSON** で動く。

### 決定木の JSON（超シンプル版）

```json
{
  "q1": {
    "text": "やりたいことは具体的ですか？",
    "yes": "q2",
    "noAction": "行動：やりたいことを30秒で具体化してみよう"
  },
  "q2": {
    "text": "理由ははっきりしていますか？",
    "yes": "q3",
    "noAction": "行動：理由を1つだけ書いてみよう"
  },
  "q3": {
    "text": "やろうとすると止まる瞬間はありますか？",
    "yes": "q4",
    "no": "q5"
  }
}
```

---

## ✨ まとめ

**Yes/No だけで"行動が止まる理由→次の一手"が分かる質問フロー**
を作ると、誰でも簡単に「次に何をすれば良いか」が見える。

質問フローは：

1. 具体性
2. 理由（価値）
3. 止まる瞬間
4. 準備の重さ
5. タスクの大きさ
6. 気持ち
7. 環境
8. 今日の一歩
